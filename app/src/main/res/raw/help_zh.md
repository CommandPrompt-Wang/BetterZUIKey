# BetterZUIKey 使用帮助

## 📖 概述

BetterZUIKey 是一个 Xposed 模块，用于接管和自定义 ZUI/ZUXOS 系统中的键盘快捷键行为。

通过本模块，你可以：
- 🚫 禁用不想要的快捷键
- 🔄 将快捷键行为改为 AOSP 原生方式
- 📱 为特定应用创建自定义快捷键模板
- ⌨️ 虚拟 Fn 键（Win 模拟 Fn → F1~F12）
- 🌍 区域差异化行为覆写（ROW/中国/韩国）
- ♿ AOSP 辅助功能开关（防抖/鼠标/粘滞/慢速键）

---

## 🎯 行为覆写模式

每个快捷键可独立选择以下行为覆写模式：

| 模式 | 说明 |
|------|------|
| **跟随系统开关** | 保持 ZUX 原始逻辑：系统开关开启则用 ZUX，关闭则透传给应用 |
| **强制 ZUX** | 强制启用 ZUX 快捷键行为，忽略系统开关状态 |
| **强制 AOSP** | 拦截 ZUX 处理，交由 Android 原生实现 |
| **透传 (OFF)** | 完全禁用该快捷键拦截，事件直接透传给前台应用 |
| **拦截消费 (BLOCK)** | 消费该快捷键事件但不执行任何操作，系统和应用都收不到 |

### 🔄 自动开启开关

修改右侧下拉菜单（行为覆写模式）时，如果左侧的**系统开关**存在、可操作且处于关闭状态，模块会**自动将其打开**。

> 💡 **建议将所有快捷键的系统开关都打开**，确保模块能正常拦截和处理这些快捷键。如果某个快捷键的系统开关未开启，其右侧下拉菜单的覆写修改**可能无效**——因为 ZUI 系统层根本不会将该快捷键事件交给模块处理。

**例外（不会自动打开开关）：**

| 快捷键 | 原因 |
|--------|------|
| `Win + Alt + 3`（防抖键） | 开关直接读写 `Settings.Secure`，不走 Config |
| `Win + Alt + 4`（鼠标键） | 同上 |
| `Win + Alt + 5`（粘滞键） | 同上 |
| `Win + Alt + 6`（慢速键） | 同上 |
| Ctrl 卡片 | 开关控制"Ctrl 长按"，下拉菜单控制"Ctrl + /"，两者功能独立 |

## ⚙️ 设置说明

### 🔌 模块总开关

全局总开关，关闭后整个模块的所有功能失效，所有快捷键恢复 ZUI 原始行为，等同于未安装模块。

> ⚠️ **注意**：切换后需要重启系统界面或重启设备才能完全生效。

### ⌨️ 虚拟 Fn 键

对于没有物理 Fn 键的键盘，本模块通过软件模拟 Fn 键功能，将键盘顶行按键映射为 F1~F12。

❗注意❗：此程序和 [KeyMapper](https://github.com/keymapperorg/KeyMapper/releases) 的“专家模式”有兼容性问题，专家模式下依然会读取按键的扫描码而非 keyCode，为防止误解特别在此说明。

 - 不过，KeyMapper 的专家模式无法正确读取和映射 ZUI 键盘的特殊按键（如静音、分屏等），因此建议使用本模块内置的虚拟 Fn 键功能来实现 F1~F12 映射，确保兼容性和稳定性。
 - 如果确实需要使用 KeyMapper 将映射后的 F1~F12 进一步映射为其他功能，请尝试“普通模式”。**但不保证可以正常工作**。

#### 两种触发模式：

**🔒 FnLock 模式**
- 按下 `` Win + ` ``（Tab 上方重音符键）切换 Fn 锁定状态
- 锁定后，单按顶行键直接输出 F1~F12
- `` Win + ` `` 被完全拦截，不会触发其他功能
- 再次按下 `` Win + ` `` 可关闭 Fn 锁定

**⌨️ 临时 Fn 模式**
- FnLock 未开启时，按住 `Win` 再按顶行键，临时输出 F1~F12
- 反过来当 FnLock 开启时，按住 `Win` 再按顶行键，临时输出原始按键功能（如音量、亮度等）

#### 默认映射（Lenovo Yoga Keyboard 17ef:6271）

| 顶行键 | Fn 映射 | 顶行键 | Fn 映射 |
|--------|---------|--------|---------|
| ESC | F1 | 亮度+ | F7 |
| 静音 | F2 | 截图 | F8 |
| 音量- | F3 | 最大化 | F9 |
| 音量+ | F4 | 分屏 | F10 |
| 闭麦 | F5 | 星星(自定义) | F11 |
| 亮度- | F6 | 删除 | F12 |

#### ⚠️ 关于系统辅助键重映射

Android 系统设置中提供了"辅助键重映射"功能（位于 设置 → 物理键盘 → 辅助键），允许将 Caps Lock、Ctrl、Meta、Alt、ESC 等按键互相交换。

内置默认配置文件**基于 ESC（keyCode 111）映射为 F1**。如果你在系统设置中开启了辅助键重映射（例如将 ESC 映射为 BACK），默认的 Fn 配置文件将无法正确匹配 ESC 键，虚拟 Fn 功能会失效。

**解决方案**：
- **推荐**：进入 [键盘检测](app://keyboarddetect) 页面，在当前重映射状态下重新录制一份键盘配置，然后导入使用
- 或者：关闭系统辅助键重映射，使用模块内置默认配置

> 💡 重映射状态由 `/data/system/input-manager-state.xml` 定义，模块读取 keyCode 进行匹配，因此重映射后 keyCode 与配置文件不再对应。

#### 键盘配置文件

不同键盘的物理按键扫描码不同，模块内置了默认的 Lenovo Yoga Keyboard (17ef:6271) 映射表。对于其他键盘型号，可以通过制作配置文件或者导入 JSON 配置文件来定义自定义的扫描码映射。

**配置步骤：**
1. 进入 [虚拟 Fn 键](app://fnsettings) 设置页面
2. 选择或导入匹配的键盘配置文件
3. 可通过 [配置文件管理](app://profilemanage) 查看、导入、删除配置

### 🌍 区域覆写

ZUI 系统根据两个属性决定快捷键的差异化行为：
- `ro.config.lgsi.region` — 判定中国/海外 (ROW)
- `ro.config.lgsi.countrycode` — 判定韩国 (KR)

#### 区域 (region)

| 选项 | 说明 |
|------|------|
| 默认 | 不干预，使用设备原始值 |
| 中国 (CN) | 强制中国区，Meta 按住连续切语言 |
| 国际 (ROW) | 强制海外区，Ctrl+Shift/Alt+Shift 切输入法 |
| 自定义 | 输入任意区域代码（如 "TW"、"JP"） |

#### 国家/地区 (countrycode)

| 选项 | 说明 |
|------|------|
| 默认 | 不干预，使用设备原始值 |
| 韩国 (KR) | 强制韩国区，启用 Alt_RIGHT 切韩语 |
| 自定义 | 输入任意国家代码 |

> ⚠️ 修改区域可能导致部分快捷键行为异常，请确认理解各区域差异后再修改。

## 📋 应用模板

应用模板允许你为特定前台应用覆写快捷键行为。当匹配的应用处于前台时，模板中定义的快捷键覆写将**覆盖**全局设置；模板中未列出的快捷键则继续继承「快捷键」Tab 中的全局设置。

> 💡 如果同一个应用匹配了多个模板，**列表中的第一个**启用模板生效。

### 使用场景举例：
- 🎮 在游戏中禁用 Win 键，防止误触跳出
- 📺 在视频播放器中屏蔽 Print Screen 截屏快捷键
- 🎯 在特定应用中自定义 App1/App2 按键行为

### 操作指南：

**创建模板**
1. 切换到「应用模板」Tab
2. 点击底部 **「新建模板」** 按钮
3. 输入模板名称（如「游戏模式」「视频模式」）

**绑定应用**
1. 点击模板卡片上的 **「选择应用」**
2. 在应用列表中搜索并勾选目标应用（支持多选）
3. 同一应用不能被多个模板同时绑定，选择时会提示冲突
4. 点击「完成」确认

**编辑快捷键覆写**
1. 点击模板卡片进入编辑器
2. 列表中显示全部 48 个快捷键，每个键可独立设置：
   - **开关**：启用或禁用该快捷键
   - **行为模式**：跟随系统 / 强制 ZUI / 强制 AOSP / 透传 / 拦截消费
   - **继承全局**：选择此项则回退到全局设置
3. 被覆写的键会以蓝色标记显示
4. 返回时自动保存

**管理模板**
- ✏️ **重命名**：点击模板卡片上的「重命名」
- 🔛 **启用/禁用**：通过模板卡片右侧开关控制
- 🗑️ **删除**：点击「删除模板」，确认后移除

> ⚠️ 修改模板后，当目标应用切换到前台时即刻生效。无需重启。

> 1. 本模块需要 **Xposed 框架**（LSPosed 等）激活才能生效，请在 LSPosed 管理器中勾选本模块并选择「**系统框架**」作用域（注意：是"系统框架"而非"Android 系统"）。
>
> 2. 修改快捷键行为可能导致系统功能异常或快捷键冲突，请务必阅读完本文档后再进行修改。
>
> 3. 部分快捷键在锁屏状态或特定场景下无效，这是 ZUI 系统的底层限制，非模块问题。
>
> 4. 区域覆写仅对 ZUI/ZUXOS 系统有效。
>
> 5. 虚拟 Fn 键通过 `Win + `` ` 切换 Fn 锁定；锁定后顶行键直接输出 F1~F12（无需按 Win）。FnLock 关闭时也可通过 Win 临时触发。
>
> 6. OneVision 特性需要设备硬件和系统版本支持，部分旧款设备或海外版可能不支持。
>
> 7. 如果修改后键盘行为异常，请先将模块总开关关闭，重启设备后恢复原始行为，再逐项排查问题配置。

## 📎 附录

### 附录 A — 制作键盘配置文件

有两种方式：

**方法一：半自动制作（推荐）**

1. 进入「[虚拟 Fn 键](app://fnsettings) → [制作配置文件](app://keyboarddetect)」
2. 按任意键自动检测键盘 VID:PID
3. 逐一按下顶行 12 个按键，每次按下后点击「填入」将当前按键的 keyCode 填入选中的 F-key 位置
  - 填入后，会自动切换到下一个 F-key 以便继续填入
  - 如果不慎填错，可以重新选中 F-key 再按下正确的按键进行覆盖填入
4. 点击「复制配置模板」获取填充好的 JSON 模板文本，可以直接用于导入。
   - 可自行修改 `name` 和 `friendlyName` 字段来定义键盘名称和显示名称
   - 如果存在未填入的 F-key，对应位置会保留“【请填充】”占位符。

**方法二：手工编写**

按以下格式编写 JSON 文件，然后通过「[配置文件管理](app://profilemanage) → 导入」加载：

```json
{
  "profiles": {
    "<vendor:product>": {
      "name": "键盘名称（内部标识）",
      "friendlyName": "显示名称（用户可见）",
      "keys": {
        "F1":  { "keyCode": <Android keyCode> },
        "F2":  { "keyCode": <Android keyCode> }
      }
    }
  }
}
```

**示例**（联想小新Pad Pro GT 磁吸键盘，`17ef:6271`）：

```json
{
  "profiles": {
    "17ef:6271": {
      "name": "Lenovo Keyboard Pack For Yoga Keyboard",
      "friendlyName": "联想小新平板Pro GT 磁吸键盘",
      "keys": {
        "F1":  { "keyCode": 1 },
        "F2":  { "keyCode": 501 },
        "F3":  { "keyCode": 25 },
        "F4":  { "keyCode": 24 }
      }
    }
  }
}
```

**字段说明：**
- `<vendor:product>` — 键盘的 USB VID:PID（十六进制小写），即型号标识（非序列号）
- `keys` — F1~F12 对应的物理按键，使用 `keyCode`（Android KeyEvent keyCode，非 Linux 键码）
- Fn 映射触发时模块注入对应 F-key（F1=131, F2=132, ..., F12=142）

> 💡 keyCode 可通过「制作配置文件」页面实时查看，也可在 LSPosed 日志中查找 `ALL KEYS keyCode=`。

### 附录 B — 区域覆写的系统属性说明

模块通过 Hook `SystemProperties.get()` 覆写以下属性：
- `ro.config.lgsi.region` — 区域代码
- `ro.config.lgsi.countrycode` — 国家代码

**ZUI 的判定逻辑：**

| 模式 | 条件 |
|------|------|
| 中国模式 | `value` 为空 或 等于 "CN" |
| ROW（海外）模式 | 非空且不等于 "CN" |
| 韩国模式 | `countrycode` 转大写后等于 "KR" |

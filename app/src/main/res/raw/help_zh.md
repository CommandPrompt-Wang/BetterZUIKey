# BetterZUIKey 使用帮助

## 📖 概述

BetterZUIKey 是一个 Xposed 模块，用于接管和自定义 ZUI/ZUXOS 系统中的键盘快捷键行为。

通过本模块，你可以：
- 🚫 禁用不想要的快捷键
- 🔄 将快捷键行为改为 AOSP 原生方式（以绕过埋点统计）
- 📱 为特定应用创建自定义快捷键模板
- ⌨️ 虚拟 Fn 键（Win 模拟 Fn → F1~F12）
- 🔤 输入法增强（切换输入法/语言绑定物理组合键）
- ♿ AOSP 辅助功能开关（防抖/鼠标/粘滞/慢速键）
- 🔍 在快捷键列表中按名称、开关状态、覆写模式搜索和筛选

## 🏠 主页

主页顶部显示 **状态卡片**，反映模块运行状态：

| 颜色 | 状态 | 含义 |
|------|------|------|
| 🟢绿色 | 已激活 + 已 Root | 模块正确作用于系统框架，且已获得 Root 权限 |
| 🟠橙色 | 已激活，无 Root | 作用域正确但无 Root 权限（部分功能受限） |
| 🟡黄色 | 作用域错误 | 模块已加载但不在 system_server 中<br />请在 LSPosed 中将作用域设为「系统框架」 |
| 🔴红色 | 未激活 | 模块未激活 — 请检查 LSPosed 管理器 |

其它特性：

- **下拉刷新**：在主页向下滑动可重新检测模块状态和 Root 权限
- **点击状态卡片**：尝试通过打开 LSPosed 管理器
- **黄色警告条**：出现错误时底部弹出，可复制错误信息

## ⌨️ 快捷键

「快捷键」页面列出所有支持的快捷键，每个快捷键可独立配置开关和覆写模式。完整快捷键参考见[附录 A](#-附录-a--快捷键参考)。

### 🔍 搜索与筛选

- **搜索**：用空格或 `+` 分隔多个关键词实现 AND 搜索。例如 `Win + Ctrl` 匹配同时包含 "Win" 和 "Ctrl" 的快捷键。
- **筛选**：通过筛选下拉菜单，可按**系统开关状态**（开/关/无开关）和**覆写模式**（保持默认/启用 ZUX 实现/启用 AOSP 实现/关闭/忽略）组合过滤快捷键列表。


### 🎯 行为覆写模式

每个快捷键可独立选择以下行为覆写模式：

| 模式 | 说明 |
|------|------|
| **保持默认** | 保持 ZUX 原始逻辑：系统开关开启则用 ZUX，关闭则透传给应用 |
| **启用 ZUX 实现** | 启用 ZUX 快捷键行为，忽略系统开关状态 |
| **启用 AOSP 实现** | 拦截 ZUX 处理，交由 Android 原生实现 |
| **关闭** | 完全禁用该快捷键拦截，事件直接透传给前台应用 |
| **忽略** | 消费该快捷键事件但不执行任何操作，系统和应用都收不到 |

#### 🔄 自动开启开关

修改右侧下拉菜单（行为覆写模式）时，如果左侧的**系统开关**存在、可操作且处于关闭状态，模块会**自动将其打开**。

> 💡 **建议将所有快捷键的系统开关都打开**，确保模块能正常拦截和处理这些快捷键。如果某个快捷键的系统开关未开启，其右侧下拉菜单的覆写修改**可能无效**——因为 ZUI 系统层根本不会将该快捷键事件交给模块处理。

**例外（不会自动打开开关）：**

| 快捷键 | 原因 |
|--------|------|
| `Win + Alt + 3`（防抖键） | 开关是功能开关（直接启用/禁用辅助功能本身），不是快捷键开关 |
| `Win + Alt + 4`（鼠标键） | 同上 |
| `Win + Alt + 5`（粘滞键） | 同上 |
| `Win + Alt + 6`（慢速键） | 同上 |
| Ctrl 卡片 | 开关控制"Ctrl 长按"，下拉菜单控制"Ctrl + /"，两者功能独立 |

**特别地，对两个智能键（507和508）：**

不使用上表五档，而是独立三档：

| 模式 | 短按 | 长按 |
|------|------|------|
| **跟随系统** | 透传 ZUI | 透传 ZUI（系统快捷键设置） |
| **忽略** | 无动作 | 无动作 |
| **执行命令…** | 运行脚本 | 打开命令编辑器 |

长按**卡片**可跳转系统「键盘应用功能」对应快捷键设置页。

**特别地，Win 长按（Meta 长按）：**

同样提供独立选项（非上表五档）：

| 模式 | 说明 |
|------|------|
| **保持默认** | 透传，由系统决定（含语音助手等；不支持的长按可能无效果） |
| **启用 ZUX 实现** | 模块接管语音助手 |
| **忽略** | 无动作 |
| **执行命令…** | 长按 Win（≥2s）运行脚本；选中后或**长按卡片**可打开命令编辑器 |

输入态下若输入法增强将 Win 长按绑定为切换键，则优先走 IME 切换（500ms），非输入态才适用上表。

**特别地，Ctrl + Enter：**

也使用独立选项（非上表五档）：

| 模式 | 说明 |
|------|------|
| **跟随系统** | 透传，由 ZUI 系统决定（当系统开关关闭时，QQ 前台会被拦截以防误触发送） |
| **插入换行** | 拦截 Ctrl+Enter，通过输入连接向应用提交 `\n` 换行符（IME 未激活时降级为注入纯 Enter 键） |
| **透传** | 无条件放行给前台应用 |
| **忽略** | 消费事件，系统和应用都收不到 |

> ⚠ **注意**：「插入换行」在 QQ 中仍需启用 QQ 设置中的「回车键发送消息」——QQ 在输入连接层面将 `\n` 强制转为发送，模块无法绕过。

## 📋 应用模板

应用模板允许你为特定前台应用覆写快捷键行为。当匹配的应用处于前台时，模板中定义的快捷键覆写将**覆盖**全局设置；模板中未列出的快捷键则继续继承「快捷键」Tab 中的全局设置。

> 💡 如果同一个应用匹配了多个模板，**列表中的第一个**启用模板生效。

### 使用场景举例：
- 在远程中透传 Win+Tab，使远程（而不是本地）收到窗口切换指令
- 在浏览器中关闭 Ctrl+Shift+T 切换触摸板的快捷键，使浏览器收到 Ctrl+Shift+T 并恢复标签页

### 操作指南：

**创建模板**
1. 切换到「应用模板」Tab
2. 点击底部 **「新建模板」** 按钮
3. 输入模板名称
   - 重复的名称会被重命名

**绑定应用**
1. 点击模板卡片上的 **「选择应用」**
2. 在应用列表中搜索并勾选目标应用（支持多选）
3. 同一应用不建议被多个模板同时绑定
   - 应用会提示冲突
4. 点击「完成」确认

**编辑快捷键覆写**
1. 点击模板卡片进入编辑器
2. 列表中显示全部快捷键，每个键可独立设置：
   - **开关**：启用或禁用该快捷键
   - **行为模式**：保持默认 / 启用 ZUX 实现 / 启用 AOSP 实现 / 关闭 / 忽略
   - **继承全局**：选择此项则回退到全局设置
3. 被覆写的键会以蓝色标记显示
   - 编辑器同样支持**搜索和筛选**功能
4. 返回时自动保存

**管理模板**
- 名称重复时自动添加后缀（-2, -3, …）
- 复制模板时可选择同时复制应用列表；同一应用匹配多个模板时，仅列表中第一个启用模板生效
- 长按拖动卡片可重新排序

> ⚠️ 修改模板后，当目标应用切换到前台时即刻生效。无需重启。

## ⚙️ 设置说明

### 🔌 模块总开关

全局总开关，关闭后整个模块的所有功能失效，所有快捷键恢复 ZUI 原始行为，等同于未安装模块。

### ⌨️ 虚拟 Fn 键

对于没有物理 Fn 键的键盘，本模块通过软件模拟 Fn 键功能，将键盘顶行按键映射为 F1~F12。

❗注意❗：此程序和 [KeyMapper](https://github.com/keymapperorg/KeyMapper/releases) 的"专家模式"有兼容性问题，专家模式下依然会读取按键的扫描码而非 keyCode，为防止误解特别在此说明。

 - 不过，KeyMapper 的专家模式无法正确读取和映射 ZUI 键盘的特殊按键（如静音、分屏等），因此建议使用本模块内置的虚拟 Fn 键功能来实现 F1~F12 映射，确保兼容性和稳定性。
 - 如果确实需要使用 KeyMapper 将映射后的 F1~F12 进一步映射为其他功能，请尝试"普通模式"。**但不保证可以正常工作**。

#### 触发模式：

**🔒 FnLock 模式**
- 按下 `` Win + ` ``（Tab 上方重音符键）切换 Fn 锁定状态
- 锁定后，单按顶行键直接输出 F1~F12
- `` Win + ` `` 被完全拦截，不会触发其他功能
- 再次按下 `` Win + ` `` 可关闭 Fn 锁定

**⌨️ 临时 Fn 模式**
- FnLock 未开启时，按住 `Win` 再按顶行键，临时输出 F1~F12
- 反过来当 FnLock 开启时，按住 `Win` 再按顶行键，临时输出原始按键功能（如音量、亮度等）

#### 默认映射（Lenovo Yoga Keyboard 17ef:6271）

| 顶行键 | Fn 映射 | 顶行键 | Fn 映射 |
|--------|---------|--------|---------|
| ESC | F1 | 亮度+ | F7 |
| 静音 | F2 | 截图 | F8 |
| 音量- | F3 | 最大化 | F9 |
| 音量+ | F4 | 分屏 | F10 |
| 闭麦 | F5 | 星星(自定义) | F11 |
| 亮度- | F6 | 删除 | F12 |

#### ⚠️ 关于系统辅助键重映射

Android 系统设置中提供了"辅助键重映射"功能（位于 设置 → 物理键盘 → 辅助键），允许将 Caps Lock、Ctrl、Meta、Alt、ESC 等按键互相交换。

内置默认配置文件**基于 ESC（keyCode 111）映射为 F1**。如果你在系统设置中开启了辅助键重映射（例如将 ESC 映射为 BACK），默认的 Fn 配置文件将无法正确匹配 ESC 键，虚拟 Fn 功能会失效。

**解决方案**：
- **推荐**：进入 [键盘检测](app://keyboarddetect) 页面，在当前重映射状态下重新录制一份键盘配置，然后导入使用
- 或者：关闭系统辅助键重映射，使用模块内置默认配置

> 💡 重映射状态由 `/data/system/input-manager-state.xml` 定义，模块读取 keyCode 进行匹配，因此重映射后 keyCode 与配置文件不再对应。

#### 键盘配置文件

不同键盘的物理按键扫描码不同，模块内置了默认的 Lenovo Yoga Keyboard (17ef:6271) 映射表。对于其他键盘型号，可以通过制作配置文件或者导入 JSON 配置文件来定义自定义的扫描码映射。

**配置步骤：**
1. 进入 [虚拟 Fn 键](app://fnsettings) 设置页面
2. 选择或导入匹配的键盘配置文件
3. 可通过 [配置文件管理](app://profilemanage) 查看、导入、删除配置

详细的制作方法见[附录 B](#-附录-b--制作键盘配置文件)。

### 🔤 输入法增强

在设置标签页点击「输入法增强」进入配置页。可分别指定「切换输入法」和「切换语言」各绑定一个物理组合键：

| 选项 | 说明 |
|------|------|
| 保持默认 | 不干预，ZUI 原生处理 |
| Ctrl+Shift | 仅 ROW 产品可用 |
| Ctrl+Space | 通用组合键 |
| Alt+Shift | 仅 ROW 产品可用 |
| 右 Alt | 独立 Alt_RIGHT 按键 |
| Win 长按 | 输入态下 Win 长按触发，非输入态由语音助手接管 |
| 关闭 | 阻止 ZUI 执行切换，按键本身透传 |
| 忽略 | 强制消费，App 和系统均不接收 |

输入态下 Win 长按优先走 IME 切换（500ms 定时），非输入态走语音助手（2s 定时）。

#### IME Profile

模块通过 JSON profile 自动匹配当前输入法执行切换策略。三种策略：

| 策略 | 原理 | 适用场景 |
|------|------|---------|
| `framework` | 反射调用 `switchToNextInputMethodSubtype()` | 声明了 subtype 的输入法（如 GBoard） |
| `hook` | 在 IME 进程中 hook 内部切换方法 | 需自行反编译获取类名与方法签名，作者不提供 |
| `keyremap` | 注入指定组合键给 IME | 通用兜底 |

Profile 为 JSON 文件，通过「输入法增强 → 管理 Profiles」导入。每个 profile 绑定一个输入法包名；未匹配时不做任何操作，由模块原逻辑处理。

## ⚠️ 注意事项

1. 本模块需要 LSPosed 2.0.x+，请在 LSPosed 管理器中勾选本模块并选择推荐的「**系统框架**」作用域

2. 修改快捷键行为可能导致系统功能异常或快捷键冲突，请务必理解相应效果后再进行修改。

3. 部分快捷键在锁屏状态或特定场景下无效，这是 ZUI 系统的底层限制，非模块问题。

4. 如果修改后键盘行为异常，请先将模块总开关关闭，重启设备后恢复原始行为，再逐项排查问题配置。

| `Win + Alt + 6` | 慢速键 (Slow Keys) | 按键需按住一定时间才识别 |

> 💡 AOSP 辅助键通过 `Settings.Secure` 控制，与系统辅助功能设置同步。开关直接读写系统数据库。

## 附录 B — 制作键盘配置文件

有两种方式：

**方法一：半自动制作（推荐）**

1. 进入「[虚拟 Fn 键](app://fnsettings) → [制作配置文件](app://keyboarddetect)」
2. 按任意键自动检测键盘 VID:PID
3. 逐一按下顶行 12 个按键，每次按下后点击「填入」将当前按键的 keyCode 填入选中的 F-key 位置
  - 填入后，会自动切换到下一个 F-key 以便继续填入
  - 如果不慎填错，可以重新选中 F-key 再按下正确的按键进行覆盖填入
4. 点击「复制配置模板」获取填充好的 JSON 模板文本，可以直接用于导入。
   - 可自行修改 `name` 和 `friendlyName` 字段来定义键盘名称和显示名称
   - 如果存在未填入的 F-key，对应位置会保留"【请填充】"占位符。

**方法二：手工编写**

按以下格式编写 JSON 文件，然后通过「[配置文件管理](app://profilemanage) → 导入」加载：

```json
{
  "profiles": {
    "<vendor:product>": {
      "name": "键盘名称（内部标识）",
      "friendlyName": "显示名称（用户可见）",
      "keys": {
        "F1":  { "keyCode": <Android keyCode> },
        "F2":  { "keyCode": <Android keyCode> }
      }
    }
  }
}
```

**示例**（联想小新Pad Pro GT 磁吸键盘，`17ef:6271`）：

```json
{
  "profiles": {
    "17ef:6271": {
      "name": "Lenovo Keyboard Pack For Yoga Keyboard",
      "friendlyName": "联想小新平板Pro GT 磁吸键盘",
      "keys": {
        "F1":  { "keyCode": 1 },
        "F2":  { "keyCode": 501 },
        "F3":  { "keyCode": 25 },
        "F4":  { "keyCode": 24 }
      }
    }
  }
}
```

**字段说明：**
- `<vendor:product>` — 键盘的 USB VID:PID（十六进制小写），即型号标识（非序列号）
- `keys` — F1~F12 对应的物理按键，使用 `keyCode`（Android KeyEvent keyCode，非 Linux 键码）
- Fn 映射触发时模块注入对应 F-key（F1=131, F2=132, ..., F12=142）

> 💡 keyCode 可通过「制作配置文件」页面实时查看，也可在 LSPosed 日志中查找 `ALL KEYS keyCode=`。
